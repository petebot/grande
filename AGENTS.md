# Project agent guidance

## Design-system requirements

Before making UI changes:

1. Read `DESIGN_SYSTEM_CONTRACT.md` completely.
2. Read `docs/design-system.md` completely.
3. Inspect existing tokens and relevant components before adding new ones.
4. Follow the contract's component, accessibility, documentation, and validation requirements.
5. Update the project design-system document and `/system` specimen when the system changes.
6. Report exceptions and unverified behavior explicitly.

The storefront-sign reconstruction and sampled colors are provisional references. Do not represent
them as approved production brand assets without written confirmation from Grande Burrito.
